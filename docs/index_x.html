Services - (Controllers)

In JamilX, "Services" are the building blocks of an application.

A Service is a PHP class that extends the JXService class and implements the JXServiceI interface. Services are responsible for processing incoming requests, interacting with the application's data sources (such as a database or external API), and returning the appropriate response. Each Service is registered with JamilX, allowing it to be accessed via a unique URL. Requests to a Service URL are handled by JamilX's routing system, which determines which Service should be called based on the request's URL parameters. Once the correct Service has been identified, JamilX instantiates the corresponding class and calls its methods to process the request. Services are designed to be modular and reusable, allowing developers to easily build complex applications by combining and reusing existing Services. JamilX includes a number of built-in Services that provide common functionality, such as authentication and user management, and developers can also create their own custom Services to meet the specific needs of their application.
